Gap Identificado

TABELA AUSENTE: agent_skills

O banco do Secretário possui 17 tabelas, mas NENHUMA tabela para rastrear as 23 skills jurídicas do Hermes.

Isso impede:

  • Rastreabilidade de uso de skills por sessão
  • Consultas relacionais sobre skills
  • Integração automática Hermes → Secretário

Schema Proposto

CREATE TABLE agent_skills ( id UUID PRIMARY KEY DEFAULT uuid_generate_v4(), nome VARCHAR(100) UNIQUE NOT NULL, categoria VARCHAR(50), descricao TEXT, path TEXT NOT NULL, ferramentas JSONB, triggers JSONB, uso_count INT DEFAULT 0, ultima_utilizacao TIMESTAMP, ativo BOOLEAN DEFAULT TRUE ); CREATE TABLE sessao_skills ( sessao_id UUID REFERENCES sessoes(id), skill_id UUID REFERENCES agent_skills(id), resultado VARCHAR(50), duracao_segundos INT );

Benefícios da Integração

Liberação de Memória Hermes

Com o Secretário consolidando entidades relacionais, o Hermes pode manter apenas preferências essenciais (~500 bytes), delegando todas as consultas relacionais.
